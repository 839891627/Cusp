# Cusp UI 重设计 AI Brief

> 用途：把这份文档直接给其他 AI 工具（如 ChatGPT / Claude / Figma AI / v0 等），让它们基于统一上下文产出可执行的 UI 重设计方案。

## 1) 项目背景

- 产品名：`Cusp`
- 平台：`macOS` 原生桌面应用（SwiftUI）
- 产品定位：本地代理客户端，强调**清晰、稳定、日常高频操作效率**
- 当前核心能力：
  - 订阅导入与解析
  - 运行模式切换（Direct / Global / Rules）
  - 节点与策略组切换
  - 日志、测速、订阅诊断
  - 菜单栏快速操作
- 语言：英文 + 简体中文双语

## 2) 设计目标（必须满足）

1. **信息更清晰**：用户在 3 秒内看懂当前运行状态（是否连接、当前模式、当前节点/策略、异常风险）。
2. **关键操作更快**：连接/断开、模式切换、节点切换、订阅刷新路径更短，减少层级跳转。
3. **可维护性更好**：形成统一组件与视觉规范，避免页面风格割裂。
4. **高级感更强**：视觉上接近高质量 macOS 工具类产品，避免“廉价控制台感”。

## 3) 非目标（避免跑偏）

- 不改动底层代理引擎逻辑
- 不引入移动端设计
- 不做品牌重命名
- 不做过度动画或炫技交互（优先稳定与效率）

## 4) 当前 IA（信息架构）与关键模块

请基于下列模块重构结构与层级关系：

- `Overview`（总览）
- `Routes`（连接路由）
- `Strategy`（策略）
- `Nodes`（节点）
- `Rules`（规则）
- `Subscriptions`（订阅）
- `Logs`（日志）
- `Settings`（设置）

术语约束（务必保持一致）见 `Docs/UI-Terminology.md`。

## 5) 目标用户与核心场景

- 用户类型：
  - 需要稳定代理能力的日常用户
  - 有一定网络工具认知的效率型用户
- 高频任务：
  - 快速确认当前状态是否健康
  - 一键切换模式与节点
  - 刷新订阅并检查是否生效
  - 故障时快速恢复（如恢复系统代理、查看日志）

## 6) 视觉与交互方向

### 视觉关键词

- 专业、克制、现代、可信赖
- 轻量层次、清晰分区、状态优先

### 交互原则

- 关键状态常驻可见
- 高频动作就近触达
- 危险操作有明确确认与反馈
- 文案简短，避免术语堆叠

## 7) 平台与实现约束

- 必须符合 `macOS` 使用习惯（窗口、侧边栏、工具栏、菜单栏联动）
- 需要考虑浅色/深色模式一致性
- 双语文本长度差异下布局不破版
- 不依赖复杂 3D、重度动效或高成本素材

## 8) 交付要求（让 AI 输出这些）

请输出以下内容（按顺序）：

1. **重构后的信息架构图**（页面与模块关系）
2. **关键用户流程**（连接、切换模式、切换节点、刷新订阅、故障恢复）
3. **低保真线框**（至少：Overview、Strategy/Nodes、Subscriptions、Settings）
4. **高保真视觉方案**（含浅色/深色）
5. **组件系统草案**（按钮、状态标签、列表项、卡片、表单、反馈）
6. **设计 Token 建议**（颜色、间距、圆角、阴影、字号层级）
7. **可落地建议**（哪些优先做，哪些可第二阶段）

## 9) 验收标准（评估设计好坏）

- 新用户能在短时间判断“现在是否正常工作”
- 高频操作步骤明显减少
- 全局视觉一致，模块间无突兀风格跳变
- 错误/异常状态比旧版更易理解和恢复
- 设计可映射到 SwiftUI 组件，不是纯概念稿
